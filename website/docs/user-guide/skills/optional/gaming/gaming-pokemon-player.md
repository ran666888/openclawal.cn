## 何时使用

## 启动流程

### 1. 首次设置（克隆、venv、安装）
该存储库是 GitHub 上的 NousResearch/pokemon-agent。然后克隆它
设置Python 3.10+虚拟环境。使用 uv（为了速度首选）
创建 venv 并使用以下命令以可编辑模式安装包
pyboy额外。如果 uv 不可用，请回退到 python3 -m venv + pip。

在此计算机上，它已设置在 /home/teknium/pokemon-agent
准备好 venv — 只需 cd 并源 .venv/bin/activate。

您还需要一个 ROM 文件。向用户询问他们的。在这台机器上
其中一个存在于该目录内的 roms/pokemon_red.gb 中。
切勿下载或提供 ROM 文件 - 始终询问用户。

### 2.启动游戏服务器
从激活了 venv 的 pokemon-agent 目录中，运行
pokemon-agent 使用 --rom 指向 ROM 和 --port 9876 来服务。
使用 & 在后台运行它。
要从已保存的游戏恢复，请添加 --load-state 和保存名称。
等待 4 秒启动，然后使用 GET /health 进行验证。

### 3. 设置实时仪表板供用户观看
通过 localhost.run 使用 SSH 反向隧道，以便用户可以查看
浏览器中的仪表板。用ssh连接，转发本地
端口 9876 到 nokey@localhost.run 上的远程端口 80。重定向输出
到日志文件，等待 10 秒，然后 grep 日志中的 .lhr.life
网址。向用户提供附加了 /dashboard/ 的 URL。
隧道 URL 每次都会更改 — 如果重新启动，则为用户提供新的隧道 URL。

## 保存和加载

### 何时保存
- 每 15-20 轮游戏
- 总是在健身房战斗、对手遭遇或危险的战斗之前
- 进入新城镇或地牢之前
- 在您不确定采取任何行动之前

### 如何保存
使用描述性名称 POST /save。好的例子：
before_brock、route1_start、mt_moon_entrance、got_cut

### 如何加载
使用保存名称 POST /load。

### 列出可用的保存
GET /saves 返回所有保存的状态。

### 在服务器启动时加载
启动服务器时使用 --load-state 标志自动加载保存。
这比启动后通过 API 加载要快。

## 操作参考
- press_a — 确认、通话、选择
- press_b — 取消，关闭菜单
- press_start — 打开游戏菜单
- walk_up/down/left/right — 移动一个方块
-hold_b_N — 按住 B N 帧（用于加速文本）
- wait_60 — 等待大约 1 秒（60 帧）
- a_until_dialog_end — 重复按 A 直至对话框清除

## 经验中的重要提示

### 不断运用愿景
- 每移动 2-4 步就截取一次屏幕截图
- RAM状态告诉你位置和生命值，但不告诉你周围是什么
- 壁架、栅栏、标志、建筑门、NPC — 仅通过屏幕截图可见
- 向视觉模型提出具体问题：“我以北的一块瓷砖是什么？”
- 当卡住时，请在尝试随机方向之前先截图

### 扭曲过渡需要额外的等待时间
当穿过门或楼梯时，屏幕会逐渐变黑
地图过渡。您必须等待它完成。添加2-3 wait_60
任何门/楼梯变形后的操作。无需等待，位置就会显示
过时了，你会认为你仍然在旧地图中。

### 建筑出口陷阱
当您退出建筑物时，您会直接出现在门前。
如果你向北走，你就会回到里面。总是先回避
向左或向右走 2 格，然后朝您想要的方向前进。

### 对话框处理
第一代文本逐个字母缓慢滚动。为了加快对话速度，
按住 B 120 帧，然后按 A。根据需要重复。持有 B 使
以最大速度显示文本。然后按 A 前进到下一行。
a_until_dialog_end 操作检查 RAM 对话框标志，但该标志
不捕获所有文本状态。如果对话框似乎卡住，请使用手册
相反，按住 b + 按 a 模式并通过屏幕截图进行验证。

### 壁架是单向的
壁架（小悬崖边缘）只能向下（向南）跳，不能攀爬
向上（北）。如果被向北的壁架挡住，你必须向左或向右走
找到它周围的间隙。使用视觉来识别方向
加普群岛明确询问视觉模型。

### 导航策略
- 一次移动 2-4 步，然后截图检查位置
- 进入新区域时，立即截图定位
- 询问视觉模型“到[目的地]的方向是什么？”
- 如果尝试 3 次以上仍卡住，请截屏并完全重新评估
- 不要垃圾邮件 10-15 个动作 - 你会过度或陷入困境

### 逃离狂野的战斗
在战斗菜单上，RUN位于右下角。从默认达到它
光标位置（FIGHT，左上角）：按下然后向右移动光标
运行，然后按 A。用hold_b 换行以加快文本/动画的速度。

### 战斗（战斗）
在战斗菜单“战斗”位于左上角（默认光标位置）。
按 A 进入移动选择，再次按 A 使用第一个移动。
然后按住 B 加速攻击动画和文本。

## 战斗策略

### 决策树
1、想抓吗？ → 削弱然后扔精灵球
2.狂野你不需要吗？ → 运行
3、类型优势？ → 使用超级有效的招式
4、没有优势？ → 使用最强的 STAB 招式
5.生命值低？ → 切换或使用药水

### 第一代类型表（关键对决）
- 水胜火、土、岩石
- 火击败草、虫、冰
- 草胜过水、地、石
- 电击水、飞
- 地面击败火、电、岩石、毒药
- 心灵击败战斗、毒药（在第一代中占主导地位！）

### 第一代怪癖
- 特殊属性 = 特殊动作的进攻和防守
- 心灵类型被压制（幽灵动作被窃听）
- 基于速度统计的致命一击
- 包裹/束缚防止对手行动
- 焦点能量错误：降低暴击率而不是提高它

## 停止播放
1. 通过 POST /save 使用描述性名称保存游戏
2. 使用 PKM:PROGRESS 更新内存
3. 告诉用户：“游戏已保存为[名称]！说‘玩神奇宝贝’即可继续。”
4. 终止服务器和隧道后台进程

## 陷阱
- 切勿下载或提供 ROM 文件
- 在未检查视力的情况下，请勿发送超过 4-5 个动作
- 离开建筑物后向北走之前一定要避开
- 门/楼梯扭曲后始终添加 wait_60 x2-3
- 通过 RAM 进行的对话框检测不可靠 — 使用屏幕截图进行验证
