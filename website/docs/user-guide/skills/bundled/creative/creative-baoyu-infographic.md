# 信息图生成器

改编自 OpenClaw 工具生态的 [baoyu-infographic](https://github.com/JimLiu/baoyu-skills)。

两个维度：**布局**（信息结构）×**风格**（视觉美观）。自由组合任何布局和任何风格。

当用户要求创建信息图、视觉摘要、信息图或使用“信息图”、“可视化”或“高密度信息大图”等术语时触发此技能。用户提供内容（文本、文件路径、URL 或主题），并可选择指定布局、样式、宽高比或语言。

## 核心原则

- 忠实地保留源数据 - 无需摘要或改写（但**在包含在输出中之前剥离任何凭证、API 密钥、令牌或秘密**）
- 在构建内容之前定义学习目标
- 视觉传达结构（标题、标签、视觉元素）

## 工作流程

### 第 1 步：分析内容

**加载参考文献**：阅读本技能中的“references/analysis-framework.md”。

1.保存源内容（文件路径或使用“write_file”粘贴→“source.md”）
   - **备份规则**：如果`source.md`存在，则重命名为`source-backup-YYYYMMDD-HHMMSS.md`
2. 分析：主题、数据类型、复杂性、基调、受众
3. 检测源语言和用户语言
4. 从用户输入中提取设计指令
5. 将分析保存到“analysis.md”
   - **备份规则**：如果`analysis.md`存在，则重命名为`analysis-backup-YYYYMMDD-HHMMSS.md`

详细格式请参见“references/analysis-framework.md”。

### 第 2 步：生成结构化内容 → `structured-content.md`

将内容转换为信息图结构：
1. 标题和学习目标
2. 部分：关键概念、内容（逐字）、视觉元素、文本标签
3. 数据点（所有统计数据/引用均准确复制）
4. 用户设计说明

**规则**：仅限 Markdown。没有新信息。忠实地保存数据。从输出中删除任何凭据或秘密。

有关详细格式，请参阅“references/structured-content-template.md”。

### 第 3 步：推荐组合

**3.1 首先检查关键字快捷方式**：如果用户输入与 **关键字快捷方式** 表中的关键字匹配，则自动选择关联的布局并将关联的样式优先考虑为首要推荐。跳过基于内容的布局推断。

**3.2 否则**，根据以下条件推荐 3-5 种布局×样式组合：
- 数据结构→匹配布局
- 内容基调→风格匹配
- 观众期望
- 用户设计说明

### 第 4 步：确认选项

使用“澄清”工具与用户确认选项。由于“澄清”一次处理一个问题，因此首先问最重要的问题：

**Q1 — 组合**：展示 3+ 布局×风格组合并说明理由。要求用户选择一个。

**Q2 — 宽高比**：询问宽高比偏好（横向/纵向/方形或自定义宽：高）。

**问题 3 — 语言**（仅当源 ≠ 用户语言时）：询问文本内容应使用哪种语言。

### 步骤 5：生成提示 → `prompts/infographic.md`

**备份规则**：如果`prompts/infographic.md`存在，则重命名为`prompts/infographic-backup-YYYYMMDD-HHMMSS.md`

**加载引用**：从“references/layouts/<layout>.md”读取选定的布局，从“references/styles/<style>.md”读取样式。

结合：
1. 布局定义来自 `references/layouts/<layout>.md`
2. 样式定义来自 `references/styles/<style>.md`
3. 来自 `references/base-prompt.md` 的基本模板
4. 第 2 步中的结构化内容
5. 所有文本均采用确认语言

**“{{ASPECT_RATIO}}”的宽高比分辨率**：
- 命名预设→比例字符串：横向→`16:9`，纵向→`9:16`，方形→`1:1`
- 自定义宽高比 → 按原样使用（例如，`3:4`、`4:3`、`2.35:1`）

使用“write_file”将组装的提示保存到“prompts/infographic.md”。

### 第 6 步：生成图像

使用“image_generate”工具和步骤 5 中组装的提示。

- 将宽高比映射为 image_generate 的格式：`16:9` → `landscape`、`9:16` → `portrait`、`1:1` → `square`
- 对于自定义比率，选择最接近的命名方面
- 失败时，自动重试一次
- 将生成的图像 URL/路径保存到输出目录

### 步骤 7：输出摘要

报告：主题、布局、风格、方面、语言、输出路径、创建的文件。

## 陷阱

1. **数据完整性至关重要** — 切勿总结、解释或更改源统计数据。 “73%的增长”必须是“73%的增长”，而不是“大幅增长”。
2. **剥离机密** — 在包含在任何输出文件中之前，始终扫描源内容中的 API 密钥、令牌或凭据。
3. **每个部分一条消息** - 每个信息图部分都应传达一个清晰的概念。重载部分会降低可读性。
4. **样式一致性** — 参考文件中的样式定义必须在整个信息图中一致应用。不要混合风格。
5. **image_generate 宽高比** — 该工具仅支持“landscape”、“portrait”和“square”。像“3:4”这样的自定义比例应该映射到最近的选项（在这种情况下为纵向）。
