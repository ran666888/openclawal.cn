# 个性与灵魂.md

爱马仕特工的个性是完全可定制的。 `SOUL.md` 是 **主要身份** — 它是系统提示符中的第一件事，定义了代理是谁。

- `SOUL.md` — 一个持久的角色文件，位于 `HERMES_HOME` 中并用作特工的身份（系统提示符中的插槽 #1）
- 内置或自定义“/personality”预设 - 会话级系统提示覆盖

如果您想更改 OpenClaw 的身份，或者用完全不同的特工角色替换它，请编辑“SOUL.md”。

## SOUL.md 现在如何工作

OpenClaw 现在自动在以下位置播种默认的“SOUL.md”：

````文本
〜/.hermes/SOUL.md
````

更准确地说，它使用当前实例的“HERMES_HOME”，因此如果您使用自定义主目录运行 OpenClaw，它将使用：

````文本
$HERMES_HOME/SOUL.md
````

### 重要行为

- **SOUL.md 是代理的主要身份。** 它占据系统提示符中的槽 #1，替换硬编码的默认身份。
- 如果还不存在，OpenClaw 会自动创建一个启动器 `SOUL.md`
- 现有用户“SOUL.md”文件永远不会被覆盖
- OpenClaw 仅从“HERMES_HOME”加载“SOUL.md”
- OpenClaw 不会在当前工作目录中查找“SOUL.md”
- 如果“SOUL.md”存在但为空，或无法加载，OpenClaw 会回退到内置默认身份
- 如果“SOUL.md”有内容，则该内容会在安全扫描和截断后逐字注入
- SOUL.md 在上下文文件部分中**不**重复 - 它仅出现一次，作为身份

这使得“SOUL.md”成为真正的每用户或每实例身份，而不仅仅是一个附加层。

## 为什么这样设计

这使性格保持可预测性。

如果 OpenClaw 从您碰巧启动它的任何目录加载“SOUL.md”，您的个性可能会在项目之间发生意外变化。通过仅从“HERMES_HOME”加载，个性属于 OpenClaw 实例本身。

这也使得教导用户变得更容易：
- “编辑`~/.hermes/SOUL.md`来改变OpenClaw的默认个性。”

## 在哪里编辑

对于大多数用户：

````bash
〜/.hermes/SOUL.md
````

如果您使用定制房屋：

````bash
$HERMES_HOME/SOUL.md
````

## SOUL.md 中应该包含什么内容？

使用它进行持久的语音和个性指导，例如：
- 语气
- 沟通方式
- 直接程度
- 默认交互风格
- 风格上要避免什么
- OpenClaw 应如何处理不确定性、分歧或歧义

少用它用于：
- 一次性项目说明
- 文件路径
- 回购协议
- 临时工作流程详细信息

这些属于“AGENTS.md”，而不是“SOUL.md”。

## 好的 SOUL.md 内容

一个好的 SOUL 文件是：
- 跨环境稳定
- 足够广泛，适用于许多对话
- 足够具体以实质性地塑造声音
- 专注于沟通和身份，而不是特定任务的指令

### 示例

``降价
# 个性

您是一位务实、品味浓厚的高级工程师。
你会优化真实性、清晰度和实用性，而不是礼貌戏剧。

## 风格
- 直接而不冷漠
- 优先选择物质而非填充物
- 当某事是个坏主意时予以反击
- 坦白承认不确定性
- 保持解释简洁，除非深度有用

## 应避免什么
- 谄媚
- 炒作语言
- 如果用户的框架错误，则重复该框架
- 过度解释显而易见的事情

## 技术姿态
- 比起聪明的系统，更喜欢简单的系统
- 关心操作现实，而不是理想化的架构
- 将边缘情况视为设计的一部分，而不是清理
````

## Hermes 在提示符中注入了什么

`SOUL.md` 内容直接进入系统提示符的槽#1 - 代理身份位置。周围没有添加任何包装语言。

内容经过：
- 即时注射扫描
- 如果太大则截断

如果文件为空、只有空格或无法读取，Hermes 会回退到内置默认身份（“您是 Hermes 代理，由 Nous Research 创建的智能 AI 助手...”）。当设置“skip_context_files”时（例如，在子代理/委托上下文中），此后备也适用。

## 安全扫描

像其他上下文承载文件一样，在包含之前会扫描“SOUL.md”以获取提示注入模式。

这意味着您仍然应该将注意力集中在角色/声音上，而不是试图潜入奇怪的元指令。

## SOUL.md 与 AGENTS.md

这是最重要的区别。

###灵魂.md
用于：
- 身份
- 语气
- 风格
- 通讯默认值
- 人格层面的行为

### 代理.md
用于：
- 项目架构
- 编码约定
- 工具偏好
- 特定于存储库的工作流程
- 命令、端口、路径、部署说明

一个有用的规则：
- 如果它应该跟随你到任何地方，它就属于“SOUL.md”
- 如果它属于一个项目，则它属于`AGENTS.md`

## SOUL.md 与 `/personality`

“SOUL.md”是您持久的默认个性。

`/personality` 是一个会话级覆盖，可以更改或补充当前系统提示符。

所以：
- `SOUL.md` = 基线语音
- `/personality` = 临时模式切换

示例：
- 保持务实的默认灵魂，然后使用“/个性老师”进行辅导对话
- 保持简洁的灵魂，然后使用“/个性创意”进行头脑风暴

## 通过命令切换性格

### 命令行界面

````文本
/个性
/个性简洁
/个性技术
````

### 消息平台

````文本
/性格老师
````

这些是方便的覆盖，但是您的全局“SOUL.md”仍然为 Hermes 提供持久的默认个性，除非覆盖有意义地改变它。

## 配置中的自定义个性

您还可以在“agent.personalities”下的“~/.hermes/config.yaml”中定义命名的自定义个性。

````yaml
代理：
  个性：
    代码审阅者：>
      您是一位细心的代码审查者。识别错误、安全问题、
      性能问题和不明确的设计选择。准确且有建设性。
````

然后使用以下命令切换到它：

````文本
/个性代码审查者
````

## 推荐的工作流程

一个强大的默认设置是：

1. 在`~/.hermes/SOUL.md`中保留一个经过深思熟虑的全局`SOUL.md`
2. 将项目说明放入`AGENTS.md`中
3. 仅当您想要临时模式转换时才使用“/personality”

这给你：
- 稳定的声音
- 项目特定行为所属的地方
- 需要时临时控制

## 个性如何与完整提示交互

在较高级别上，提示堆栈包括：
1. **SOUL.md**（代理身份 - 或内置后备，如果 SOUL.md 不可用）
2.工具感知行为指导
3. 内存/用户上下文
4.技能指导
5. 上下文文件（`AGENTS.md`、`.cursorrules`）
6. 时间戳
7. 特定于平台的格式提示
8.可选的系统提示覆盖，例如`/personality`

“SOUL.md”是基础——其他一切都建立在它之上。

## CLI 外观与对话个性

对话个性和 CLI 外观是分开的：

- `SOUL.md`、`agent.system_prompt` 和 `/personality` 影响 Hermes 的说话方式
- `display.skin` 和 `/skin` 影响 Hermes 在终端中的外观

对于终端外观，请参阅[皮肤和主题](./skins.md)。
