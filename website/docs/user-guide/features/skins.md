# 皮肤和主题

皮肤控制 Hermes CLI 的 **视觉呈现**：横幅颜色、旋转器面孔和动词、响应框标签、品牌文本和工具活动前缀。

对话风格和视觉风格是不同的概念：

- **个性**改变代理的语气和措辞。
- **皮肤** 更改 CLI 的外观。

## 更换皮肤

````bash
/skin # 显示当前皮肤并列出可用皮肤
/skin ares # 切换到内置皮肤
/skin mytheme # 从 ~/.hermes/skins/mytheme.yaml 切换到自定义皮肤
````

或者在`~/.hermes/config.yaml`中设置默认皮肤：

````yaml
显示：
  皮肤：默认
````

## 可配置键的完整列表

### 颜色（`颜色：`）

控制整个 CLI 中的所有颜色值。值是十六进制颜色字符串。

|关键|描述 |默认（`默认`皮肤）|
|-----|-------------|--------------------------|
| `banner_border` |启动横幅周围的面板边框 | `#CD7F32`（铜牌）|
| `banner_title` |横幅中的标题文本颜色 | `#FFD700`（金）|
| `banner_accent` |横幅中的部分标题（可用工具等）| `#FFBF00`（琥珀色）|
| `banner_dim` |横幅中的静音文本（分隔符、辅助标签）| `#B8860B`（暗黄花）|
| `横幅文本` |横幅中的正文（工具名称、技能名称）| `#FFF8DC`（玉米丝）|
| `ui_accent` |一般 UI 强调色（突出显示、活动元素）| `#FFBF00` |
| `ui_label` | UI 标签和标记 | `#4dd0e1`（青色）|
| `ui_ok` |成功指标（复选标记、完成）| `#4caf50`（绿色）|
| `ui_error` |错误指示器（失败、阻塞）| `#ef5350`（红色）|
| `ui_warn` |警告指示器（注意、批准提示）| `#ffa726`（橙色）|
| `提示` |交互提示文字颜色| `#FFF8DC` |
| `输入规则` |输入区域上方的水平线 | `#CD7F32` |
| `响应边框` |代理响应框周围的边框（ANSI 转义）| `#FFD700` |
| `会话标签` |会话标签颜色 | `#DAA520` |
| `会话边界` |会话 ID 边框颜色变暗 | `#8B8682` |
| `状态栏_bg` | TUI 状态/使用栏的背景颜色 | `#1a1a2e` |
| `voice_status_bg` |语音模式状态徽章的背景颜色 | `#1a1a2e` |
| `选择背景` | TUI 鼠标选择荧光笔的背景颜色。未设置时回退到“completion_menu_current_bg”。 | `#333355` |
| `completion_menu_bg` |完成菜单列表的背景颜色 | `#1a1a2e` |
| `completion_menu_current_bg` |活动完成行的背景颜色 | `#333355` |
| `completion_menu_meta_bg` |完成元列的背景颜色 | `#1a1a2e` |
| `completion_menu_meta_current_bg` |活动完成元列的背景颜色 | `#333355` |

### 旋转器（`旋转器：`）

控制等待 API 响应时显示的动画微调器。

|关键|类型 |描述 |示例|
|-----|------|-------------|---------|
| `等待面孔` |字符串列表 |等待 API 响应时循环面孔 | `["(⚔)", "(⛨)", "(▲)"]` |
| `思考面` |字符串列表 |模型推理期间循环的面孔 | `["(⚔)", "(⌁)", "(<>)"]` |
| `思考动词` |字符串列表 |微调消息中显示的动词 | `[“锻造”、“策划”、“锤击计划”]` |
| '翅膀' | [左，右] 对列表 |旋转器周围的装饰支架| `[["⟪⚔", "⚔⟫"], ["⟪▲", "▲⟫"]]` |

当微调器值为空时（例如“default”和“mono”），将使用“display.py”中的硬编码默认值。

### 品牌（`品牌：`）

整个 CLI 界面中使用的文本字符串。

|关键|描述 |默认 |
|-----|-------------|---------|
| `代理名称` |横幅标题和状态显示中显示的名称 | 《爱马仕特工》|
| `欢迎` | CLI 启动时显示欢迎消息 | “欢迎来到爱马仕代理商！输入您的消息或 /help 命令。` |
| `再见` |退出时显示消息 | '再见！ ⚕` |
| `响应标签` |响应框标题上的标签 | ` ⚕ 爱马仕 ` |
| `提示符号` |用户输入提示之前的符号（裸标记，渲染器添加尾随空格）| `❯` |
| `help_header` | `/help` 命令输出的标题文本 | `(^_^)?可用命令` |

### 其他顶级键

|关键|类型 |描述 |默认 |
|-----|------|-------------|---------|
| `工具前缀` |字符串| CLI 中工具输出行的前缀字符 | `┊` |
| `工具_表情符号` |字典 |每个工具的表情符号覆盖旋转器和进度 (`{tool_name: emoji}`) | `{}` |
| `banner_logo` |字符串|丰富标记 ASCII 艺术徽标（替换默认的 HERMES_AGENT 横幅）| `""` |
| `横幅英雄` |字符串|丰富标记的英雄艺术（替换默认的杖艺术）| `""` |

## 自定义皮肤

在 `~/.hermes/skins/` 下创建 YAML 文件。用户皮肤继承了内置“默认”皮肤的缺失值，因此您只需指定要更改的键即可。

### 完全自定义皮肤 YAML 模板

````yaml
# ~/.hermes/skins/mytheme.yaml
# 完整的皮肤模板 - 显示所有按键。删除不需要的；
# 缺失值自动从“默认”皮肤继承。

名称：我的主题
描述：我的自定义主题

颜色：
  横幅边框：“#CD7F32”
  横幅标题：“#FFD700”
  横幅口音：“#FFBF00”
  banner_dim：“#B8860B”
  横幅文本：“#FFF8DC”
  ui_accent：“#FFBF00”
  ui_label：“#4dd0e1”
  ui_ok: "#4caf50"
  用户界面错误：“#ef5350”
  ui_warn：“#ffa726”
  提示：“#FFF8DC”
  输入规则：“#CD7F32”
  响应边框：“#FFD700”
  会话标签：“#DAA520”
  会话边界：“#8B8682”
  status_bar_bg: "#1a1a2e"
  voice_status_bg: "#1a1a2e"
  Selection_bg: "#333355"
  完成菜单_bg：“#1a1a2e”
  完成_菜单_当前_bg：“＃333355”
  完成_菜单_元_bg：“#1a1a2e”
  completion_menu_meta_current_bg: "#333355"

微调器：
  等待面孔：
    - “(⚔)”
    - “(⛨)”
    - “(▲)”
  思维面：
    - “(⚔)”
    - “(⌁)”
    - “(<>)”
  思考动词：
    - “处理”
    - “分析”
    - “计算”
    - “评估”
  翅膀：
    - [“⟪⚡”，“⚡⟫”]
    - [“⟪●”，“●⟫”]

品牌推广：
  agent_name：“我的经纪人”
  欢迎：“欢迎来到我的代理！输入您的消息或 /help 以获取命令。”
  再见：“再见！⚡”
  response_label：“⚡我的经纪人”
  提示符号：“⚡”
  help_header: "(⚡) 可用命令"

工具前缀：“┊”

# 每个工具的表情符号覆盖（可选）
工具表情符号：
  终端：“⚔”
  网络搜索：“🔮”
  读取文件：“📄”

# 自定义 ASCII 艺术横幅（可选，支持丰富标记）
# 横幅标志: |
# [粗体#FFD700]我的代理人[/]
# 横幅英雄: |
# [#FFD700] 这里定制艺术[/]
````

### 最小的自定义皮肤示例

由于所有内容都继承自“default”，因此最小的皮肤只需要更改不同的内容：

````yaml
名称：赛博朋克
描述：霓虹终端主题

颜色：
  横幅边框：“#FF00FF”
  横幅标题：“#00FFFF”
  横幅口音：“#FF1493”

微调器：
  Thinking_verbs：[“插入”、“解密”、“上传”]
  翅膀：
    - [“⟨⚡”，“⚡⟩”]

品牌推广：
  agent_name：“网络代理”
  response_label：“⚡网络”

工具前缀：“▏”
````

## OpenClaw Mod — 可视化皮肤编辑器

[OpenClaw Mod](https://github.com/cocktailpeanut/hermes-mod) 是一个社区构建的 Web UI，用于直观地创建和管理皮肤。您无需手动编写 YAML，而是获得具有实时预览功能的点击式编辑器。

![OpenClaw Mod 皮肤编辑器](https://raw.githubusercontent.com/cocktailpeanut/hermes-mod/master/nous.png)

**它的作用：**

- 列出所有内置和自定义皮肤
- 在可视化编辑器中打开任何皮肤，其中包含所有 OpenClaw 皮肤字段（颜色、微调器、品牌、工具前缀、工具表情符号）
- 从文本提示生成“banner_logo”文本艺术
- 将上传的图像（PNG、JPG、GIF、WEBP）转换为具有多种渲染样式（盲文、ASCII 斜坡、块、点）的“banner_hero”ASCII 艺术作品
- 直接保存到`~/.hermes/skins/`
- 通过更新`~/.hermes/config.yaml`激活皮肤
- 显示生成的 YAML 和实时预览

### 安装

**选项 1 — Pinokio（一键点击）：**

在[pinokio.computer](https://pinokio.computer)上找到它并一键安装。

**选项 2 — npx（从终端最快）：**

````bash
npx -y Hermes-mod
````

**选项 3 — 手动：**

````bash
git 克隆 https://github.com/cocktailpeanut/hermes-mod.git
cd Hermes-mod/应用程序
npm 安装
npm 启动
````

### 用法

1. 启动应用程序（通过 Pinokio 或终端）。
2. 打开**皮肤工作室**。
3. 选择要编辑的内置或自定义皮肤。
4. 从文本生成徽标和/或上传英雄艺术图像。选择渲染样式和宽度。
5. 编辑颜色、微调器、品牌和其他字段。
6.点击**保存**将皮肤YAML写入`~/.hermes/skins/`。
7. 单击“**激活**”将其设置为当前皮肤（更新“config.yaml”中的“display.skin”）。

OpenClaw Mod 尊重 `HERMES_HOME` 环境变量，因此它也适用于 [profiles](/user-guide/profiles)。

## 操作注意事项

- 内置皮肤从“hermes_cli/skin_engine.py”加载。
- 未知的皮肤会自动回退到“默认”。
- `/skin` 立即更新当前会话的活动 CLI 主题。
- `~/.hermes/skins/` 中的用户皮肤优先于同名的内置皮肤。
- 通过“/skin”进行的皮肤更改仅限会话。要使皮肤成为永久默认皮肤，请在“config.yaml”中设置它。
- `banner_logo` 和 `banner_hero` 字段支持彩色 ASCII 艺术的丰富控制台标记（例如，`[bold #FF0000]text[/]`）。
