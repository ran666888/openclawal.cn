# 宠物

OpenClaw 可以展示一个动画**宠物**——一个小的吉祥物精灵，它会对什么做出反应
代理正在做（空闲、运行工具、思考、完成、失败）
**CLI**、**TUI** 和 **桌面应用程序**。宠物来自公众
[petdex](https://github.com/crafter-station/petdex) 画廊。

宠物纯粹是装饰性的。它们对提示缓存、令牌或
代理的行为**——精灵只是一个显示问题。特点是
**默认关闭**并保持休眠状态，直到您安装并选择宠物。

## 它是如何工作的

- 宠物被安装到您的个人资料的“pets/”目录中
  (`<HERMES_HOME>/pets/<slug>/`)，因此每个 [profile](../profiles.md) 都会保留其
  自己的一套。
- 选择宠物会将 `display.pet.slug` 和 `display.pet.enabled` 写入
  `config.yaml` — 没有任何内容被存储为秘密或环境变量。
- 每个表面都会监视它已经跟踪的活动并将其映射到六个表面之一
  动画状态。贴图位于一个地方，因此每个表面的行为都相同
  相同：

  |代理活动 |宠物状态|
  | --- | --- |
  |工具/转动刚刚失败 | `失败` |
  |计划已完成（所有待办事项已完成）| ‘跳跃’（庆祝）|
  |干净利落地完成一个转弯 | `波` |
  |一个工具正在执行 | `运行` |
  |该模型正在思考/阅读 | `审查` |
  |飞行中转弯（未指定）| `运行` |
  |阻止您（澄清/批准提示已打开）| `waiting`（在传统 8 行表上回退到 `idle`）|
  |什么也没发生| `空闲` |

## 渲染

在终端 (CLI/TUI) 中，当您
终端支持图形协议（**kitty**、**Ghostty**、**WezTerm**、
**iTerm2** 或 **sixel**）。否则它会自动回退到真彩色
Unicode **半块**渲染。在管道或重定向（无 TTY）、终端内
根据设计禁用渲染。

桌面应用程序将宠物绘制为画布上的浮动精灵并切换它
从**设置→外观**。

## 快速入门 (CLI)

````bash
# 浏览图库（按子字符串过滤）
爱马仕宠物清单
爱马仕宠物清单 猫

# 安装一只宠物并让它一步激活
爱马仕宠物安装boba --select

# 在终端中预览/设置动画（Ctrl+C 停止）
爱马仕宠物展

# 检查你的设置
爱马仕宠物医生
````

## `hermes pets` 命令

|目标|命令|
| --- | --- |
|浏览画廊 | `hermes 宠物列表 [查询] [--limit N]` |
|列出已安装的宠物 | `爱马仕宠物列表--已安装` |
|安装宠物 | `hermes pets install <slug> [--select] [--force]` |
|设置活跃宠物| `hermes pets select [slug]`（省略选择器的 slug）|
|随处调整宠物大小 | `hermes pets scale <factor>`（例如`0.5`，限制在 0.1–3.0）|
|预览/动画 | `hermes pets show [slug] [--state <s>] [--cycle] [--once] [--mode <m>] [--scale <f>]` |
|禁用宠物 | “爱马仕宠物关闭”|
|删除已安装的宠物 | `爱马仕宠物移除 <slug>` |
|诊断设置 | 『爱马仕宠物医生』 |

“爱马仕宠物展”标志：

- `--state` — 播放单个状态（`idle`、`wave`、`run`、`failed`、`review`、
  ‘跳’)。
- `--cycle` — 循环遍历每个状态。
- `--once` — 播放一次而不是循环播放。
- `--mode` — 覆盖渲染协议（`kitty`、`iterm`、`sixel`、
  `unicode`、`auto`)。
- `--scale` — 覆盖屏幕上的比例（`0` = 使用配置）。

## `/pet` 斜线命令

在 CLI 和 TUI 中，您可以在不离开会话的情况下管理宠物：

- `/pet` — 打开/关闭宠物（如果没有，则采用第一个安装的宠物）
  活跃）。
- `/pet list` — 浏览图库。
- `/pet scale <factor>` — 调整宠物各处的大小（例如 `/pet scale 0.5`）。
- `/pet <slug>` — 收养特定的宠物。
- `/pet off` — 禁用宠物。

在 TUI 中，“/pet list”打开交互式选择器覆盖层；在桌面应用程序中
它会打开 Cmd+K 宠物调色板。

## 桌面应用程序

在桌面应用程序中，您可以通过两种方式管理宠物：

- **Cmd+K →“宠物...”** — 无需离开即可浏览、搜索、领养和切换宠物
  键盘（镜像主题选择器）。
- **设置 → 外观** — 同一个图库加上一个 **尺寸滑块**
  当您拖动时实时调整浮动吉祥物的大小。

两者都采用/切换/调整浮动吉祥物的大小 - 尺寸更改适用
立即；收养一只新宠物很快就会点燃它。

### 弹出覆盖

**按住 Shift 键并单击**浮动宠物，将其弹出到自己的透明状态中，
始终在最上面的桌面窗口。当赫尔墨斯在外面时，它仍然可见
最小化（Codex 风格），因此一眼就能看出代理正在做什么。

弹出后的手势：

|手势|行动|
| --- | --- |
| **拖动** |将宠物移动到屏幕上的任何位置，甚至在应用程序之外。它的点和输入/输出状态在重新启动后仍然存在。 |
| **单击** |打开迷你编辑器即可向最近的会话发送提示 - 无需显示应用程序。 |
| **双击** |切换应用程序窗口：如果它在前面，则最小化它；如果它隐藏，则恢复它。 |
| **按住 Shift 键并单击** |把宠物放回窗户里。 |
| **邮件图标** |仅当您离开时回合结束时才会出现；单击以在最新线程上引发应用程序（并将其标记为已读）。 |

只有弹出的宠物才会显示**语音气泡**（“工作……”、“思考……”、
“轮到你了”，...) — 在窗口中，应用程序本身就是表面，因此宠物会停留在
那里很安静。

覆盖层是应用内宠物的纯粹傀儡 - 它没有单独的网关
连接，并且永远不会出现在扩展坞或应用程序切换器中。

## 配置

所有设置都位于“config.yaml”中的“display.pet”下：

````yaml
显示：
  宠物：
    enabled: false # 主开/关（选择宠物后为 true）
    slug: "" # 活跃宠物;空 = 首次安装
    渲染模式：自动#自动|小猫|项目 |六色 |统一码 |关闭
    比例：0.33 # 主尺寸旋钮（相对于原生 192x208 帧）
    unicode_cols: 0 # 终端宽度的硬覆盖（0 = 从比例导出）
````

- **`scale`** 是单个主尺寸旋钮。一个数字会缩小每个表面：
  桌面画布通过它缩放像素，CLI/TUI 派生它们的像素
  终端列宽度。半块回退限制了易读性
  地板 - 它不能像真正的像素小猫/GUI 渲染那样缩小，而无需
  变成糊状，所以同样的“鳞片”在小猫下面看起来很脆，但被踩在了地板上
  半块。
- **`render_mode: auto`** 检测到 kitty/iTerm2/sixel 并回退到 unicode
  半块。显式设置它以强制协议或“off”以禁用
  终端渲染，同时将宠物保持在桌面上。
- **`unicode_cols`** 独立于`scale`固定终端列宽度；
  将其保留为“0”以从“scale”导出宽度。

## 故障排除

运行“爱马仕宠物医生”——它报道说：

- pets 目录以及安装了哪些 pets，
- `display.pet.enabled`、`display.pet.slug` 和已解析的活动宠物，
- 配置的“render_mode”、检测到的终端图形协议以及
  TTY 的有效模式，
- Pillow（用于精灵解码）是否可导入。

一旦安装、选择、启用宠物，并且 Pillow 已启动，它会打印“✓ 就绪”
可用。

常见问题：

- 宠物仅在**安装并选择**后才会显示（“启用：true”）。
- 在管道/重定向（无 TTY）内，终端渲染在设计上被禁用。
- petdex npm CLI 安装到 `~/.codex/pets`；爱马仕用的是自己的
  相反，配置文件范围为“<HERMES_HOME>/pets/”——通过“hermes pets”安装。
