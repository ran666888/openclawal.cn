#WeCom（企业微信）

将 OpenClaw 连接至腾讯的企业消息平台【WeCom】(https://work.weixin.qq.com/) (企业微信)。该适配器使用 WeCom 的 AI Bot WebSocket 网关进行实时双向通信 - 无需公共端点或 Webhook。

另请参阅：[WeCom Callback](./wecom-callback.md) 了解入站 Webhook 设置。

## 先决条件

- WeCom 组织帐户
- 在 WeCom 管理控制台中创建的 AI 机器人
- 机器人凭据页面中的机器人 ID 和秘密
- Python 包：`aiohttp` 和 `httpx`

## 设置

### 第 1 步：创建人工智能机器人

#### 推荐：扫描到创建（一个命令）

````bash
Hermes网关设置
````

选择 **WeCom** 并使用您的 WeCom 移动应用程序扫描二维码。 OpenClaw 将自动创建具有正确权限的机器人应用程序并保存凭据。

设置向导将：
1. 在您的终端中显示二维码
2.等待您用WeCom手机应用程序扫描
3.自动检索Bot ID和Secret
4.指导您完成访问控制配置

#### 替代方案：手动设置

如果扫描创建不可用，向导将退回到手动输入：

1.登录【微信管理控制台】(https://work.weixin.qq.com/wework_admin/frame)
2. 导航至 **应用程序** → **创建应用程序** → **AI 机器人**
3.配置机器人名称和描述
4. 从凭据页面复制 **Bot ID** 和 **Secret**
5. 运行 `hermes gateway setup`，选择 **WeCom**，并在出现提示时输入凭据

:::警告
将机器人秘密保密。任何拥有它的人都可以冒充你的机器人。
:::

### 第2步：配置OpenClaw

#### 选项 A：交互式设置（推荐）

````bash
Hermes网关设置
````

选择 **WeCom** 并按照提示操作。该向导将引导您完成：
- 机器人凭证（通过二维码扫描或手动输入）
- 访问控制设置（白名单、配对模式或开放访问）
- 通知的主页频道

#### 选项 B：手动配置

将以下内容添加到 `~/.hermes/.env` 中：

````bash
WECOM_BOT_ID=您的机器人 ID
WECOM_SECRET=你的秘密

# 可选：限制访问
WECOM_ALLOWED_USERS=user_id_1,user_id_2

# 可选：cron/通知的主频道
WECOM_HOME_CHANNEL=chat_id
````

### 步骤 3：启动网关

````bash
爱马仕网关
````

## 特点

- **WebSocket 传输** — 持久连接，不需要公共端点
- **DM 和群组消息传送** — 可配置的访问策略
- **每组发件人白名单** — 细粒度控制谁可以在每个组中进行交互
- **媒体支持** — 图像、文件、语音、视频上传和下载
- **AES 加密媒体** — 入站附件自动解密
- **引用上下文** — 保留回复线程
- **Markdown 渲染** — 富文本响应
- **回复相关性** — 响应与入站消息上下文相关
- **自动重新连接** — 连接断开时的指数退避

:::note 流媒体和打字指示器
WeCom 适配器将每个响应作为单个完整消息提供 - 它确实如此
**不**逐个令牌地流响应，并且它**不**显示打字
指标。 “回复关联”（如下）仅将响应线程化到其入站
请求；这不是直播。
:::

## 配置选项

在“platforms.wecom.extra”下的“config.yaml”中设置这些：

|关键|默认 |描述 |
|-----|---------|-------------|
| `bot_id` | — | WeCom AI Bot ID（必填）|
| `秘密` | — | WeCom AI Bot 秘密（必填）|
| `websocket_url` | `wss://openws.work.weixin.qq.com` | WebSocket 网关 URL |
| `dm_policy` | `打开` | DM 访问：“开放”、“允许列表”、“已禁用”、“配对” |
| `组策略` | `打开` |组访问：`open`、`allowlist`、`disabled` |
| `allow_from` | `[]` | DM 允许的用户 ID（当 dm_policy=allowlist 时）|
| `group_allow_from` | | `group_allow_from` `[]` |允许的组 ID（当 group_policy=allowlist 时）|
| `团体` | `{}` |每组配置（见下文）|

## 访问策略

### DM 政策

控制谁可以向机器人发送直接消息：

|价值|行为 |
|--------|----------|
| `打开` |任何人都可以通过 DM 机器人（默认） |
| `白名单` |只有“allow_from”中的用户 ID 可以 DM |
| `已禁用` |所有 DM 均被忽略 |
| `配对` |配对模式（用于初始设置）|

````bash
WECOM_DM_POLICY=允许名单
````

### 组策略

控制机器人在哪些组中做出响应：

|价值|行为 |
|--------|----------|
| `打开` |机器人在所有组中响应（默认）|
| `白名单` |机器人仅响应 `group_allow_from` 中列出的组 ID |
| `已禁用` |所有群组消息均被忽略 |

````bash
WECOM_GROUP_POLICY=允许名单
````

### 每组发件人白名单

为了进行细粒度控制，您可以限制允许哪些用户与特定组内的机器人进行交互。这是在`config.yaml`中配置的：

````yaml
平台：
  欢迎：
    启用：真
    额外：
      bot_id: "你的机器人 ID"
      秘密：“你的秘密”
      group_policy: “允许名单”
      组允许来自：
        - “group_id_1”
        - “group_id_2”
      团体：
        组_id_1：
          允许来自：
            - “用户_爱丽丝”
            - “用户鲍勃”
        组_id_2：
          允许来自：
            - “用户查理”
        “*”：
          允许来自：
            - “用户管理员”
````

**它是如何工作的：**

1. `group_policy` 和 `group_allow_from` 控件确定是否允许某个组。
2. 如果某个组通过了顶级检查，则“groups.<group_id>.allow_from”列表（如果存在）会进一步限制该组内的哪些发件人可以与机器人交互。
3. 通配符“*”组条目用作未明确列出的组的默认值。
4. 白名单条目支持“*”通配符允许所有用户，且条目不区分大小写。
5. 条目可以选择使用 `wecom:user:` 或 `wecom:group:` 前缀格式 - 前缀会自动删除。

如果没有为组配置“allow_from”，则允许该组中的所有用户（假设该组本身通过了顶级策略检查）。

## 媒体支持

### 入站（接收）

适配器接收来自用户的媒体附件并将其缓存在本地以供代理处理：

|类型 |如何处理 |
|------|-----------------|
| **图像** |下载并缓存在本地。支持基于 URL 和 Base64 编码的图像。 |
| **文件** |下载并缓存。文件名保留在原始消息中。 |
| **语音** |如果可用，将提取语音消息文本转录。 |
| **混合消息** | WeCom 混合类型消息（文本+图像）被解析并提取所有组件。 |

**引用的消息：** 来自引用（回复）消息的媒体也会被提取，因此代理具有有关用户正在回复的内容的上下文。

### AES 加密媒体解密

WeCom 使用 AES-256-CBC 加密一些入站媒体附件。适配器会自动处理此问题：

- 当入站媒体项包含“aeskey”字段时，适配器会下载加密字节并使用 AES-256-CBC 和 PKCS#7 填充对其进行解密。
- AES 密钥是“aeskey”字段的 base64 解码值（必须恰好为 32 字节）。
- IV 来自密钥的前 16 个字节。
- 这需要“cryptography”Python 包（“pip install cryptography”）。

无需配置 - 收到加密媒体时解密会透明地发生。

### 出站（发送）

|方法|它发送什么 |尺寸限制|
|--------|--------------|------------|
| `发送` | Markdown 短信 | 4000 个字符 |
| `发送图像` / `发送图像文件` |原生图像消息 | 10 MB |
| `发送文档` |文件附件 | 20 MB |
| `发送语音` |语音消息（AMR 格式仅适用于本机语音）| 2MB |
| `发送视频` |视频留言| 10 MB |

**分块上传：** 文件通过三步协议以 512 KB 的块上传（init → chunks → finish）。适配器会自动处理此问题。

**自动降级：** 当媒体超出本机类型的大小限制但低于 20 MB 的绝对文件限制时，它会自动作为通用文件附件发送：

- 图片 > 10 MB → 作为文件发送
- 视频 > 10 MB → 作为文件发送
- 语音 > 2 MB → 作为文件发送
- 非 AMR 音频 → 作为文件发送（WeCom 仅支持 AMR 原生语音）

超过 20 MB 绝对限制的文件将被拒绝，并向聊天发送一条信息消息。

## 回复模式响应

当机器人通过 WeCom 回调接收消息时，适配器会记住入站请求 ID。如果在请求上下文仍处于活动状态时发送响应，适配器将使用 WeCom 的回复模式 (`aibot_respond_msg`) 将响应直接与入站消息关联起来。这在 WeCom 客户端中提供了更自然的对话体验。

完整的响应作为单个消息传递——适配器不会增量地传输令牌。如果入站请求上下文已过期或不可用，适配器将回退到通过“aibot_send_msg”主动发送消息。

回复模式也适用于媒体：上传的媒体可以作为对原始消息的回复发送。

## 连接和重新连接

该适配器维护与 WeCom 网关“wss://openws.work.weixin.qq.com”的持久 WebSocket 连接。

### 连接生命周期

1. **连接：** 打开 WebSocket 连接并发送带有 bot_id 和密钥的 `aibot_subscribe` 身份验证帧。
2. **Heartbeat：** 每 30 秒发送一次应用程序级 ping 帧以保持连接处于活动状态。
3. **Listen：** 持续读取入站帧并调度消息回调。

### 重新连接行为

连接丢失时，适配器使用指数退避来重新连接：

|尝试|延迟|
|--------|--------|
|第一次重试 | 2 秒 |
|第二次重试 | 5 秒 |
|第三次重试 | 10 秒 |
|第四次重试 | 30 秒 |
|第 5 次以上重试 | 60 秒 |

每次成功重新连接后，退避计数器都会重置为零。所有挂起的请求 future 在断开连接时都会失败，因此调用者不会无限期挂起。

### 重复数据删除

使用具有 5 分钟窗口和最大缓存 1000 个条目的消息 ID 来对入站消息进行重复数据删除。这可以防止在重新连接或网络中断期间对消息进行双重处理。

## 故障排除

|问题 |修复 |
|---------|-----|
| `需要 WECOM_BOT_ID 和 WECOM_SECRET` |设置两个环境变量或在设置向导中进行配置 |
| `WeCom 启动失败：aiohttp 未安装` |安装aiohttp：`pip install aiohttp` |
| `WeCom 启动失败：httpx 未安装` |安装 httpx：`pip install httpx` |
| `无效的秘密 (errcode=40013)` |验证秘密是否与您的机器人的凭据匹配 |
| `等待订阅确认超时` |检查“openws.work.weixin.qq.com”的网络连接 |
|机器人在群组中不响应 |检查 `group_policy` 设置并确保组 ID 在 `group_allow_from` 中 |
|机器人会忽略组中的某些用户 |检查“groups”配置部分中每个组的“allow_from”列表 |
|媒体解密失败 |安装“密码学”：“pip install cryptography”|
| `WeCom 媒体解密需要加密` |入站媒体采用 AES 加密。安装：`pip install cryptography` |
|以文件形式发送语音消息 | WeCom 仅支持 AMR 格式的原生语音。其他格式会自动降级为文件。 |
| “文件太大”错误 | WeCom 对所有文件上传有 20 MB 的绝对限制。压缩或分割文件。 |
|图像作为文件发送 | > 10 MB 的图像超出了本机图像限制，并且会自动降级为文件附件。 |
| `向 WeCom 发送消息超时` | WebSocket 可能已断开连接。检查日志中是否有重新连接消息。 |
| `WeCom websocket 在身份验证期间关闭` |网络问题或凭据不正确。验证 bot_id 和机密。 |
