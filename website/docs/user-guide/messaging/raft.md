# 筏设置

OpenClaw 作为外部代理通过本地唤醒通道桥连接到 [Raft](https://raft.build)。适配器启动一个环回 HTTP 端点，该端点从桥接收无内容唤醒提示，然后将它们注入 OpenClaw 网关会话管道。代理通过 Raft CLI 读取和发送消息——适配器从不接触消息主体或传递游标。

:::info 劳动分工
- **桥**拥有：唤醒提示消耗、重复数据删除、退避、重新连接、至少一次传送和证明记录。
- **OpenClaw 适配器** 拥有：本地主机唤醒端点并向代理上下文注入简短通知。
- **代理**拥有：拉取消息（`raft message check`）、回复（`raft message send`）以及通过 CLI 进行的所有其他 Raft 交互。

该适配器不持有 Raft 凭证——仅用于桥接器和端点之间的本地主机身份验证的每会话共享令牌。
:::

---

## 先决条件

- **Raft 工作区**，您可以在其中创建外部代理
- **Raft CLI** 安装并登录到该外部代理配置文件
- **aiohttp** — Python 包（包含在 OpenClaw `[all]` extras 中）

在 Raft 中，打开 Agents 菜单，创建外部代理，然后按照设置卡安装 Raft CLI 并登录代理配置文件。创建代理后，Raft 会显示 OpenClaw 设置指南，其中包含启动网关所需的环境变量和配置。

---

## 设置

添加到`~/.hermes/.env`：

````bash
RAFT_PROFILE=您的代理配置文件
````

就是这样——当设置“RAFT_PROFILE”时，适配器会自动启用。它生成每个会话桥接令牌，选择临时端口，并在网关启动时自动生成桥接子进程。

---

## 它是如何工作的

````
Raft 服务器 → 桥接器（唤醒提示 SSE） → POST /wake → Hermes 适配器 → 代理上下文
Agent → raft 消息检查 → Raft Server（消息体）
Agent → raft 消息发送 → Raft Server（回复）
````

1. Raft 服务器通过 SSE 向桥接进程发送唤醒提示。
2. 桥接器将每个提示作为“POST /wake”转发到适配器的环回端点。
3. 适配器验证桥接令牌，验证有效负载是否不含内容，并将唤醒通知注入 OpenClaw 会话。
4. 代理看到唤醒通知并使用 Raft CLI 读取消息并回复。

唤醒有效负载**根据合约不包含任何内容** — 它们携带元数据（事件 ID、消息 ID、时间戳），但绝不携带消息正文、通道名称或发送者身份。适配器拒绝任何包含内容型字段（“text”、“body”、“content”、“messages”等）的有效负载。

---

## 桥

适配器自动生成“raft 代理桥”作为子进程，并传递端点 URL 和令牌。桥接器使用配置的配置文件连接到 Raft 服务器并开始转发唤醒提示。当网关关闭时它就会终止。

---

## 环境变量

|变量|描述 |默认 |
|----------|-------------|---------|
| `RAFT_PROFILE` | Raft 代理配置文件 slug — 设置后自动启用适配器 | _（必填）_ |
