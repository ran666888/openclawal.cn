# 注册 Microsoft Graph 应用程序

Teams 会议管道使用**仅应用程序**（守护程序）身份验证从 Microsoft Graph 读取会议记录、录音和相关工件 — 无需用户登录，每次会议无需交互同意。这需要使用管理员同意的应用程序权限注册 Azure AD 应用程序。

本指南将介绍：

1. 创建应用程序注册
2. 创建客户端密码
3. 授予管道所需的Graph API权限
4. 管理员同意这些权限
5.（可选）使用应用程序访问策略将应用程序的范围限定为特定用户

您需要**租户管理员权限**（或代表您授予同意的管理员）才能完成此操作。为您收集的值添加书签 - 它们最后会进入“~/.hermes/.env”。

## 先决条件

- 具有 Teams Premium 或 Teams 许可证的 Microsoft 365 租户，可生成会议记录和录音
- 对 Azure 门户的管理员访问权限：[entra.microsoft.com](https://entra.microsoft.com)
- 用于图形更改通知的可公开访问的 HTTPS 端点（稍后在 Webhook 侦听器步骤中设置）

## 第 1 步：创建应用程序注册

1. 以租户管理员身份登录 [entra.microsoft.com](https://entra.microsoft.com)。
2. 导航至 **身份 → 应用程序 → 应用程序注册**。
3. 单击**新注册**。
4、填写：
   - **名称：** `Hermes Teams Meeting Pipeline`（或您认识的任何名称）。
   - **支持的帐户类型：** *仅限此组织目录中的帐户（单一租户）*。
   - **重定向 URI：** 留空 — 仅应用程序身份验证不需要。
5. 单击**注册**。

您将进入应用程序的概述页面。复制两个值：

- **应用程序（客户端）ID** → `MSGRAPH_CLIENT_ID`
- **目录（租户）ID** → `MSGRAPH_TENANT_ID`

## 第 2 步：创建客户端密钥

1. 在左侧导航栏中，打开 **证书和机密**。
2. Click **New client secret**.
3. **描述：** `hermes-graph-secret`。 **过期：** 选择一个与您的轮换政策相匹配的值（通常为 6-24 个月）。
4. 单击“**添加**”。
5. 立即复制 **Value** 列 - 它仅显示一次。该值是“MSGRAPH_CLIENT_SECRET”。

> The **Secret ID** column is not the secret. You want the **Value** column.

## 步骤3：授予Graph API权限

管道使用最小可行的应用程序权限集。仅添加您需要的内容；每一项都扩大了应用程序在租户范围内可以读取的内容。

1. In the left nav, open **API permissions**.
2. 单击 **添加权限** → **Microsoft Graph** → **应用程序权限**。
3. 添加下表中与您希望管道执行的操作相匹配的权限。
4. 添加后，单击 **授予“<您的租户>”的管理员同意**。对于每个权限，“状态”列应翻转为绿色复选标记。

### Required for transcript-first summaries

|许可 | What it lets the app do |
|------------|--------------------------|
| `OnlineMeetings.Read.All` |读取 Teams 在线会议元数据（主题、参与者、加入 URL）。 |
| `OnlineMeetingTranscript.Read.All` | Read meeting transcripts generated by Teams. |

### 记录后备所需（当成绩单不可用时）

|许可 | What it lets the app do |
|------------|--------------------------|
| `OnlineMeetingRecording.Read.All` |下载 Teams 会议录音以进行离线 STT 处理。 |
| `CallRecords.Read.All` |当仅知道加入 URL 时，从通话记录中解析会议。 |

### 出站摘要传送所需（仅限图形模式）

如果“platforms.teams.extra.delivery_mode”为“graph”，则管道将摘要发布到 Teams 频道或通过 Graph API 进行聊天。如果您使用“incoming_webhook”交付模式，请跳过这些。

|许可 | What it lets the app do |
|------------|--------------------------|
| `ChannelMessage.Send` |代表应用程序将消息发布到 Teams 频道中。 |
| `Chat.ReadWrite.All` |将消息发布到 1:1 和群聊中（仅当您将“chat_id”设置为传递目标时）。 |

### 不推荐

- 没有“.All”的“OnlineMeetings.ReadWrite.All”/“Chat.ReadWrite”——比管道需求更广泛。
- 委托权限——管道使用仅应用程序（客户端凭据）流程；如果没有用户登录，委派的权限将无法工作。

## 步骤 4：（推荐）使用应用程序访问策略确定应用程序的范围

默认情况下，“OnlineMeetings.Read.All”等应用程序权限授予应用程序对租户中**每个**会议的访问权限。 For partner demos and dev tenants that's fine;对于生产，您几乎肯定希望限制应用程序可以读取哪些用户的会议。

Microsoft 为此专门为 Teams 提供了**应用程序访问策略**。该策略是仅限 PowerShell 的表面；没有门户用户界面。

从安装并连接了 MicrosoftTeams 模块的管理 PowerShell（“Connect-MicrosoftTeams”）：

````powershell
# 创建一个作用于 Hermes 应用程序的策略
新的 CsApplicationAccessPolicy `
  -Identity "Hermes-Meeting-Pipeline-Policy" `
  -AppIds“<MSGRAPH_CLIENT_ID>”`
  -描述“将 Hermes 会议管道限制为允许列出的用户”

# 将策略授予管道可以读取其会议的特定用户
授予-CsApplicationAccessPolicy`
  -PolicyName“Hermes-Meeting-Pipeline-Policy”`
  -身份“alice@example.com”

授予-CsApplicationAccessPolicy`
  -PolicyName“Hermes-Meeting-Pipeline-Policy”`
  -身份“bob@example.com”
````

授予后传播最多可能需要 30 分钟。验证：

````powershell
测试 CsApplicationAccessPolicy -Identity“alice@example.com”-AppId“<MSGRAPH_CLIENT_ID>”
````

如果没有该策略，**任何**用户的会议都是可读的 - 这就是技术上授予的权限。不要在生产租户上跳过此步骤。

## 步骤 5：将凭据写入您的环境文件

将您收集的三个值放入`~/.hermes/.env`中：

````bash
MSGRAPH_TENANT_ID=<目录租户 ID>
MSGRAPH_CLIENT_ID=<应用程序客户端 ID>
MSGRAPH_CLIENT_SECRET=<客户端秘密值>
````

设置文件权限，以便只有您可以读取机密：

````bash
chmod 600 ~/.hermes/.env
````


成功运行会打印一个长令牌字符串和一个运行状况字典，显示“cached: True”和接近 3600 的“expires_in_seconds”值。失败会生成带有 Azure 错误代码的“MicrosoftGraphTokenError” - 最常见的是：

| Azure 错误 |意义|修复 |
|-------------|---------|-----|
| `AADSTS7000215：无效的客户端密钥` |秘密值不匹配或已过期。 |步骤2中生成新的secret；更新`.env`。 |
| `AADSTS700016：找不到应用程序` | “MSGRAPH_CLIENT_ID”错误或租户错误。 |仔细检查步骤 1 中的值是否来自同一个应用程序。 |
| `AADSTS90002：找不到租户` | “MSGRAPH_TENANT_ID”中的拼写错误。 |再次从应用程序概述中复制目录（租户）ID。 |
|调用时（而非令牌时间）的 `insufficient_claims` Token 获取但 Graph 返回 401/403。 |您跳过了第 3 步管理员同意，或添加了权限但尚未重新同意。重新访问 API 权限并再次单击“**授予管理员同意**”。 |

## 轮换客户端密钥

Azure 客户端密钥有硬性过期。在你的过期之前：

1. 在步骤 2 中创建第二个客户端密钥，而不删除第一个客户端密钥。
2. 使用新值更新“~/.hermes/.env”中的“MSGRAPH_CLIENT_SECRET”。
3. 重新启动网关，以便获取新的密钥：“hermes gateway restart”。
4. 通过上述烟雾测试进行验证。
5. 从 Azure 门户删除旧密钥。

## 后续步骤

凭据验证干净后，继续：

- **Webhook 侦听器设置** — 建立接收图形更改通知的“msgraph_webhook”网关平台。
- **管道配置** — 配置 Teams 会议管道运行时和操作员 CLI。
- **外向交付** — 将摘要传回到 Teams 频道或聊天中。

这些页面与添加相应运行时的 PR 并排放置。此凭据设置是一个独立的先决条件，可以安全地提前完成。
