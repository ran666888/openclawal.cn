# 提示和最佳实践

速效实用技巧合集，让您立即更有效地使用 OpenClaw。每个部分都针对不同的方面 - 浏览标题并跳转到相关内容。

:::tip 困惑该选择哪个型号？
运行“hermes setup --portal”——一次订阅即可获得 300 多个模型，包括 Claude、GPT-5 和 Gemini。请参阅[Nous 门户](/integrations/nous-portal)。
:::

## 获得最佳结果

### 具体说明你想要什么

模糊的提示会产生模糊的结果。不要“修复代码”，而是说“修复第 47 行 `api/handlers.py` 中的 TypeError — `process_request()` 函数从 `parse_body()` 接收到 `None`。”您提供的上下文越多，您需要的迭代就越少。

### 预先提供上下文

使用相关详细信息预先加载您的请求：文件路径、错误消息、预期行为。一条精心设计的信息胜过三轮澄清。直接粘贴错误回溯——代理可以解析它们。

### 使用上下文文件进行重复指令

如果您发现自己重复相同的说明（“使用制表符而不是空格”、“我们使用 pytest”、“API 位于 `/api/v2`”），请将它们放入 `AGENTS.md` 文件中。代理每次会话都会自动读取它——设置后零努力。

### 让代理使用它的工具

不要试图用手握住每一步。说“查找并修复失败的测试”，而不是“打开 `tests/test_foo.py`，查看第 42 行，然后...”代理具有文件搜索、终端访问和代码执行功能 - 让它探索和迭代。

### 使用复杂工作流程的技能

在编写长篇提示来解释如何做某事之前，请检查是否已经有相应的技能。输入“/skills”来浏览可用的技能，或者直接调用一个技能，如“/axolotl”或“/github-pr-workflow”。

## CLI 高级用户提示

### 多行输入

按 **Alt+Enter**、**Ctrl+J** 或 **Shift+Enter** 插入换行符而不发送。 `Shift+Enter` 仅当终端将其作为不同的击键发送时才起作用（默认情况下为 Kitty / foot / WezTerm / Ghostty；启用 Kitty 键盘协议后的 iTerm2 / Alacritty / VS Code 终端）。另外两个在每个航站楼工作​​。

### 粘贴检测

CLI 自动检测多行粘贴。只需直接粘贴代码块或错误回溯 - 它不会将每一行作为单独的消息发送。粘贴内容将被缓冲并作为一条消息发送。

### 中断和重定向

按 **Ctrl+C** 一次可中断客服人员的响应。然后，您可以输入新消息来重定向它。 2秒内双击Ctrl+C强制退出。当代理开始走上错误的道路时，这是无价的。

### 使用 `-c` 恢复会话

忘记了上次会议的一些内容吗？运行“hermes -c”以准确恢复您上次停下的位置，并恢复完整的对话历史记录。您还可以按标题恢复：“hermes -r“我的研究项目””。

### 剪贴板图像粘贴

按 **Ctrl+V** 将剪贴板中的图像直接粘贴到聊天中。该代理使用视觉来分析屏幕截图、图表、错误弹出窗口或 UI 模型 - 无需先保存到文件。

### 斜线命令自动完成

输入“/”并按 **Tab** 以查看所有可用命令。这包括内置命令（“/compress”、“/model”、“/title”）和每个已安装的技能。您不需要记住任何东西——制表符补全功能已经帮您解决了。

:::提示
使用“/verbose”循环切换工具输出显示模式：**关闭 → 新建 → 全部 → 详细**。 “全部”模式非常适合观察代理的行为；对于简单的问答来说，“off”是最干净的。
:::

## 上下文文件

### AGENTS.md：您项目的大脑

在项目根目录中创建一个包含架构决策、编码约定和特定于项目的说明的“AGENTS.md”。这会自动注入到每个会话中，因此代理始终了解您的项目规则。

``降价
# 项目背景
- 这是带有 SQLAlchemy ORM 的 FastAPI 后端
- 始终使用 async/await 进行数据库操作
- 测试进入tests/并使用pytest-asyncio
- 切勿提交 .env 文件
````

### SOUL.md：自定义个性

想要Hermes拥有稳定的默认语音吗？编辑 `~/.hermes/SOUL.md` （如果您使用自定义 Hermes home，则编辑 `$HERMES_HOME/SOUL.md`）。 Hermes 现在自动播种初始灵魂，并使用该全局文件作为实例范围的个性源。

有关完整演练，请参阅[将 SOUL.md 与 Hermes 一起使用](/guides/use-soul-with-hermes)。

``降价
#灵魂
您是一名高级后端工程师。简洁、直接。
除非询问，否则跳过解释。比起冗长的解决方案，更喜欢简洁的解决方案。
始终考虑错误处理和边缘情况。
````

使用“SOUL.md”来获得持久的个性。使用“AGENTS.md”获取特定于项目的说明。

### .cursorrules 兼容性

已经有 `.cursorrules` 或 `.cursor/rules/*.mdc` 文件？赫尔墨斯也读这些。无需重复您的编码约定 - 它们会从工作目录自动加载。

### 发现

OpenClaw 在会话启动时从当前工作目录加载顶级“AGENTS.md”。子目录“AGENTS.md”文件在工具调用期间延迟发现（通过“subdirectory_hints.py”）并注入到工具结果中 - 它们不会预先加载到系统提示符中。

:::提示
保持上下文文件的重点和简洁。每个字符都会计入您的代币预算，因为它们会被注入到每条消息中。
:::

## 记忆与技能

### 记忆与技能：什么去哪里

**记忆**用于记录事实：您的环境、偏好、项目位置以及代理了解的有关您的信息。 **技能**适用于程序：多步骤工作流程、特定于工具的说明和可重复使用的配方。使用记忆来表达“什么”，使用技能来表达“如何”。

### 何时创建技能

如果您发现某项任务需要 5 个以上的步骤并且您将再次执行该任务，请要求客服人员为其创建一项技能。说“将您刚刚执行的操作保存为一项名为‘部署暂存’的技能。”下次，只需输入“/deploy-staging”，代理就会加载完整的过程。

### 管理内存容量

内存是有意限制的（MEMORY.md 约为 2,200 个字符，USER.md 约为 1,375 个字符）。当它填满时，代理会合并条目。您可以通过说“清理您的内存”或“替换旧的 Python 3.9 注释 - 我们现在使用 3.12”来提供帮助。

### 让代理记住

在一次富有成效的会议结束后，说“下次记住这一点”，客服人员就会保存关键要点。您还可以具体说明：“保存到内存中，我们的 CI 使用 GitHub Actions 和 `deploy.yml` 工作流程。”

:::警告
内存是冻结的快照 - 在会话期间所做的更改不会出现在系统提示中，直到下一个会话开始。代理会立即写入磁盘，但提示缓存不会在会话中失效。
:::

## 性能与成本

### 不要破坏提示缓存

大多数 LLM 提供商都会缓存系统提示符前缀。如果您保持系统提示稳定（相同的上下文文件，相同的内存），会话中的后续消息将获得**缓存命中**，其成本要低得多。避免在会话中更改模型或系统提示。

### 在达到限制之前使用 /compress

长时间的会话会积累代币。当您发现响应速度变慢或被截断时，请运行“/compress”。这总结了对话历史记录，保留了关键上下文，同时大大减少了令牌数量。使用“/usage”来检查你的立场。

### 并行工作委托

需要同时研究三个主题？要求代理将“delegate_task”与并行子任务一起使用。每个子代理都在自己的上下文中独立运行，并且只返回最终摘要 - 大大减少了主要对话的令牌使用量。

### 使用execute_code进行批量操作

不要一次运行一个终端命令，而是要求代理编写一个可以同时执行所有操作的脚本。 “编写一个 Python 脚本将所有 `.jpeg` 文件重命名为 `.jpg` 并运行它”比单独重命名文件更便宜、更快。

### 选择正确的型号

使用“/model”在会话中切换模型。使用前沿模型（Claude Sonnet/Opus、GPT-4o）进行复杂的推理和架构决策。切换到更快的模型来执行格式化、重命名或样板生成等简单任务。

:::提示
定期运行“/usage”以查看您的令牌消耗情况。运行“/insights”以更广泛地了解过去 30 天内的使用模式。
:::

## 消息传递技巧

### 设置家庭频道

在您喜欢的 Telegram 或 Discord 聊天中使用“/sethome”将其指定为家庭频道。 Cron 作业结果和计划任务输出均在此处传递。没有它，代理就无处发送主动消息。

### 使用 /title 来组织会议

使用“/title auth-refactor”或“/title Research-llm-quantization”命名您的会话。使用“hermes session list”很容易找到命名会话，并使用“hermes -r“auth-refactor””恢复。未命名的会话堆积起来，变得无法区分。

### 团队访问的 DM 配对

启用 DM 配对，而不是手动收集白名单的用户 ID。当队友向机器人发送私信时，他们会获得一次性配对代码。您可以通过“爱马仕配对批准电报 XKGH5N7P”来批准它——简单而安全。

### 工具进度显示模式

使用“/verbose”来控制您看到的工具活动量。在消息传递平台中，少即是多——将其保持在“新”状态即可仅查看新工具调用。在 CLI 中，“全部”为您提供代理所做的所有操作的令人满意的实时视图。

:::提示
在消息平台上，会话会在空闲时间（默认：24 小时）后或每天凌晨 4 点自动重置。如果您需要更长的会话，请在“~/.hermes/config.yaml”中调整每个平台。
:::

## 安全

### 使用 Docker 处理不受信任的代码

使用不受信任的存储库或运行不熟悉的代码时，请使用 Docker 或 Daytona 作为终端后端。在“.env”中设置“TERMINAL_BACKEND=docker”。容器内的破坏性命令不会损害您的主机系统。

````bash
# 在你的 .env 中：
TERMINAL_BACKEND=码头工人
TERMINAL_DOCKER_IMAGE=hermes-sandbox:最新
````

### 避免 Windows 编码陷阱

在 Windows 上，某些默认编码（例如“cp125x”）无法表示所有 Unicode 字符，这可能会在测试或脚本中写入文件时导致“UnicodeEncodeError”。

- 首选使用显式 UTF-8 编码打开文件：

````蟒蛇
打开（“results.txt”，“w”，encoding =“utf-8”）作为f：
    f.write("✓ 一切顺利\n")
````

- 在 PowerShell 中，您还可以将当前会话切换为 UTF-8 以进行控制台和本机命令输出：

````powershell
$OutputEncoding = [控制台]::OutputEncoding = [Text.UTF8Encoding]::new($false)
````

这使 PowerShell 和子进程保持在 UTF-8 上，并有助于避免仅限 Windows 的故障。

### 选择“始终”之前先回顾一下

当代理触发危险命令批准（`rm -rf`、`DROP TABLE`等）时，您将获得四个选项：**一次**、**会话**、**始终**、**拒绝**。在选择“始终”之前请仔细考虑 - 它会永久允许该模式。从“会话”开始，直到您感到舒服为止。

### 命令批准是您的安全网

OpenClaw 在执行之前会根据精心策划的危险模式列表检查每个命令。这包括递归删除、SQL 删除、管道卷曲到 shell 等等。不要在生产中禁用它——它的存在有充分的理由。

:::警告
当在容器后端（Docker、Singularity、Modal、Daytona）中运行时，危险的命令检查将被**跳过**，因为容器是安全边界。确保您的容器镜像已正确锁定。
:::

### 使用消息传递机器人的白名单

切勿在具有终端访问权限的机器人上设置“GATEWAY_ALLOW_ALL_USERS=true”。始终使用特定于平台的允许列表（“TELEGRAM_ALLOWED_USERS”、“DISCORD_ALLOWED_USERS”）或 DM 配对来控制谁可以与您的代理交互。

````bash
# 推荐：每个平台明确的许可名单
TELEGRAM_ALLOWED_USERS=123456789,987654321
DISCORD_ALLOWED_USERS=123456789012345678

# 或者使用跨平台白名单
GATEWAY_ALLOWED_USERS=123456789,987654321
````
